# GenAI Course — Project

## Context

You are developing Commerce Agents, AI-powered assistants that help consumers search, compare, and purchase products directly within a chat interface. Your company builds these technologies to boost e-commerce conversion rates in the Agentic Era.

What sets you apart from competitors? While others build relevant chatbots tailored to specific businesses, you go further by providing full observability and solid KPI-based performance tracking — including real-time dashboards to monitor every agent interaction.

## Expectation

In groups of 2, build a chatbot that can search, retrieve, and purchase products.

You must:

1) Define functional and non-functional requirements
2) Define most relevant evaluation metrics for you application
3) Create an observability dashboard. Use existing tools like [Langfuse](https://langfuse.com/docs/observability/overview)
4) Create an evaluation dashboard
   Tools examples:
   - [Langfuse Evaluation](https://langfuse.com/docs/evaluation/overview) for observability and tracking
   - [MLflow](https://mlflow.org/llm-evaluation) for observability and tracking
   - [Ragas](https://docs.ragas.io/en/stable/) evaluation framework
5) Implement the payment layer using [Stripe](https://stripe.com)
- Create a Stripe test account (whatever region would work)
- Stripe docs to create a [payment page](https://docs.stripe.com/payments/advanced)
- Stripe [test card](https://docs.stripe.com/testing) numbers
- [Stripe AI tooling] to accelerate you coding of Stripe (https://github.com/stripe/ai/tree/main) (MCP and Skills)

## Evaluation
- 10-minute presentation (live demo required)
- Explain your design decisions: for example: Why k=4? Why did you choose these specific evaluation metrics?
- Present it like an company internal demo pitch: most real-world projects you'll run in companies are essentially sales pitches to promote an idea.
- Incomplete is okay: but never say "I didn't have time". If you couldn't finish everything, clearly state: "With more time, I would have done X, Y, Z. and here is how '...'". Saying "I wasn't able to do it" or "I don't have time" is not an acceptable answer in any corporate environment.
